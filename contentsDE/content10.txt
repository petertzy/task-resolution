<div data-message-author-role="assistant" data-message-id="c548fc52-59a1-4e7b-b09c-8e6130c3c991" dir="auto" class="min-h-8 text-message relative flex w-full flex-col items-end gap-2 whitespace-normal break-words text-start [.text-message+&amp;]:mt-5" data-message-model-slug="gpt-4o-mini"><div class="flex w-full flex-col gap-1 empty:hidden first:pt-[3px]"><div class="markdown prose w-full break-words dark:prose-invert light"><p data-start="0" data-end="117">Wenn ein Benutzer einen Link aus einem normalen Browser (z. B. einen Link aus den Google-Suchergebnissen) anklickt, wird die PWA (Progressive Web App) standardmäßig in einer normalen Webbrowser-Umgebung geöffnet und nicht als eigenständige Anwendung gestartet. Um dieses Verhalten klarer zu erklären, müssen wir verschiedene Situationen und Konzepte unterscheiden:</p>
<h3 data-start="119" data-end="136">1. Standardverhalten beim Klicken auf einen Link:</h3>
<p data-start="137" data-end="364">Wenn Sie in Google-Suchergebnissen oder in anderen normalen Webbrowsern (wie Chrome, Safari, Firefox, Edge usw.) auf einen Link klicken, ist das Standardverhalten des Browsers, die Webseite zu öffnen, auf die der Link verweist. <br data-start="221" data-end="224">Selbst wenn der Link auf eine PWA-Website verweist, behandelt der Browser ihn immer noch als normalen Webseitenlink.<br data-start="262" data-end="265"><strong data-start="265" data-end="273">Das Ergebnis ist:</strong> Die PWA wird in einem Browser-Tab oder -Fenster geöffnet, und Sie sehen weiterhin die Adressleiste des Browsers, Menüs, Tabs und andere Browser-Oberflächenelemente. Dies unterscheidet sich im Wesentlichen nicht davon, wie Sie jede andere normale Website besuchen; die PWA läuft immer noch im "Container" des Browsers.</p>
<h3 data-start="366" data-end="387">2. Das Konzept der "Installation" einer PWA:</h3>
<p data-start="388" data-end="565">Ein zentrales Merkmal von PWAs ist ihre Installierbarkeit, d. h. Benutzer können eine PWA aus dem Browser auf ihr Gerät "installieren" (z. B. über die Option "Zum Startbildschirm hinzufügen" im Browser-Menü).<br data-start="456" data-end="459"><strong data-start="459" data-end="469">Die Bedeutung der Installation:</strong> Das Installieren einer PWA bedeutet im Wesentlichen, die PWA-Website zur App-Liste des Geräts hinzuzufügen und ein App-Symbol auf dem Startbildschirm oder im App-Drawer zu erstellen.<br data-start="521" data-end="524"><strong data-start="524" data-end="537">So starten Sie nach der Installation:</strong> Sobald die PWA installiert ist, können Benutzer sie auf zwei Arten starten:</p>
<ul data-start="566" data-end="735">
<li data-start="566" data-end="649"><strong data-start="568" data-end="582">Über einen Browser-Link:</strong> Wenn der Benutzer die URL der PWA in der Adressleiste des Browsers eingibt oder einen Lesezeichen- oder Link aus dem Browserverlauf anklickt, wird die PWA weiterhin im Browser geöffnet (gleich wie in Fall 1).</li>
<li data-start="650" data-end="735"><strong data-start="652" data-end="669">Über das installierte App-Symbol:</strong> Nach der Installation der PWA können Benutzer das App-Symbol auf dem Startbildschirm oder in der App-Liste ihres Geräts finden. Durch Klicken auf dieses Symbol wird die PWA im "Standalone-App"-Modus gestartet.</li>
</ul>
<h3 data-start="737" data-end="764">3. Merkmale der PWA im "Standalone-App"-Modus:</h3>
<ul data-start="765" data-end="1030">
<li data-start="765" data-end="850"><strong data-start="767" data-end="778">Mehr wie eine native App:</strong> Eine PWA, die im "Standalone-App"-Modus gestartet wird, blendet in der Regel die Adressleiste des Browsers und die meisten Browser-Oberflächenelemente aus, was zu einer übersichtlicheren Benutzeroberfläche führt, die sich auf den App-Inhalt konzentriert und die Benutzererfahrung näher an einer nativen App ist.</li>
<li data-start="851" data-end="939"><strong data-start="853" data-end="871">Funktioniert möglicherweise in einem separaten Prozess:</strong> In einigen Betriebssystemen und Browsern kann eine PWA, die im Standalone-App-Modus gestartet wird, in ihrem eigenen Prozess laufen, der vom Browser-Prozess isoliert ist, was zu einer besseren Leistung und Ressourcennutzung führen kann.</li>
<li data-start="940" data-end="1030"><strong data-start="942" data-end="969">Möglicherweise mit mehr Systemberechtigungen (je nach PWA-Funktionalität):</strong> Eine nach der Installation betriebene PWA kann zusätzliche Systemberechtigungen anfordern, wie z. B. Push-Benachrichtigungen, Hintergrund-Synchronisation oder Zugriff auf Gerätehardware, abhängig von den Funktionen der PWA und der Benutzerautorisierung.</li>
</ul>
<h3 data-start="1032" data-end="1039">Zusammenfassung:</h3>
<ul data-start="1040" data-end="1246">
<li data-start="1040" data-end="1146"><strong data-start="1042" data-end="1057">Klick auf einen Link im Browser:</strong> Ob aus den Google-Suchergebnissen oder von anderen Webseiten, wenn Sie auf einen PWA-Link klicken, wird dieser in einer normalen Webbrowser-Umgebung geöffnet, im Wesentlichen als normale Webseite, die in einem Browser-Tab oder -Fenster läuft.</li>
<li data-start="1147" data-end="1246"><strong data-start="1149" data-end="1166">Für eine "Standalone-App"-Erfahrung:</strong> Wenn Sie möchten, dass die PWA in einem näher an einer nativen App liegenden Modus läuft, müssen Sie die PWA zuerst auf Ihrem Gerät installieren und dann das PWA-Symbol auf dem Startbildschirm oder in der App-Liste anklicken, um sie zu starten.</li>
</ul>
<h3 data-start="1248" data-end="1257">Einfache Erklärung:</h3>
<ul data-start="1258" data-end="1332">
<li data-start="1258" data-end="1291"><strong data-start="1260" data-end="1291">Browser-Link = Normaler Webseitenzugriff auf PWA (immer noch im Browser)</strong></li>
<li data-start="1292" data-end="1332"><strong data-start="1294" data-end="1332">Installiertes PWA-Symbol = Standalone-App-Modus PWA (mehr wie eine native App)</strong></li>
</ul>
<h3 data-start="1334" data-end="1359">Wie erhält man die "Standalone-App"-Erfahrung mit einer PWA?</h3>
<ol data-start="1360" data-end="1717">
<li data-start="1360" data-end="1419"><strong data-start="1363" data-end="1377">Besuchen Sie die PWA-Website:</strong> Zuerst besuchen Sie die PWA-Website, die Sie installieren möchten, über einen Browser (z. B. Chrome, Safari).</li>
<li data-start="1420" data-end="1555"><strong data-start="1423" data-end="1445">Finden Sie die Option "Zum Startbildschirm hinzufügen" oder "Installieren":</strong> Moderne Browser erkennen in der Regel die Installierbarkeit der PWA und bieten Optionen wie "Zum Startbildschirm hinzufügen" (Chrome, Safari) oder "Installieren [App-Name]" (Edge, Firefox). Diese Optionen befinden sich in der Regel im Browser-Menü (normalerweise in der oberen rechten Ecke bei den drei Punkten oder dem Mehr-Icon).</li>
<li data-start="1556" data-end="1631"><strong data-start="1559" data-end="1570">PWA installieren:</strong> Klicken Sie auf die Option "Zum Startbildschirm hinzufügen" oder "Installieren" und folgen Sie den Anweisungen, um die Installation abzuschließen. Dadurch wird ein PWA-App-Symbol auf Ihrem Startbildschirm oder in der App-Liste erstellt.</li>
<li data-start="1632" data-end="1717"><strong data-start="1635" data-end="1648">Starten über das App-Symbol:</strong> Nach der Installation kehren Sie zum Startbildschirm oder zur App-Liste Ihres Geräts zurück, suchen das PWA-App-Symbol und klicken darauf, um die PWA zu starten. Die PWA wird nun im "Standalone-App"-Modus geöffnet.</li>
</ol>
<h3 data-start="1719" data-end="1733">Benutzererfahrungstipps:</h3>
<ul data-start="1734" data-end="1950">
<li data-start="1734" data-end="1842"><strong data-start="1736" data-end="1753">Führen Sie den Benutzer klar zur Installation der PWA:</strong> Wenn Sie möchten, dass Benutzer die beste PWA-Erfahrung (d. h. Standalone-App-Modus) haben, führen Sie sie klar auf der Website zur Installation. Zum Beispiel können Sie eine Installationserinnerung an auffälliger Stelle auf der Website hinzufügen oder eine Installationseinführung anzeigen, wenn der Benutzer die Website zum ersten Mal besucht.</li>
<li data-start="1843" data-end="1950"><strong data-start="1845" data-end="1862">Erklären Sie die Vorteile der Installation der PWA:</strong> Erklären Sie den Benutzern, dass sie nach der Installation der PWA eine schnellere, bequemere und native-ähnlichere Erfahrung haben werden, wie Offline-Zugriff, Push-Benachrichtigungen usw. Heben Sie den Unterschied zwischen dem Starten der PWA über das App-Symbol und dem Zugriff über einen Browser-Link hervor.</li>
</ul>
<h3 data-start="1952" data-end="1959">Zusammenfassung:</h3>
<p data-start="1960" data-end="2028" data-is-last-node="" data-is-only-node="">Beim Klicken auf einen PWA-Link im Browser wird die PWA immer in der Browserumgebung ausgeführt. Nur wenn die PWA installiert und über das App-Symbol gestartet wird, können Benutzer eine Erfahrung haben, die näher an einer nativen App liegt.</p>
